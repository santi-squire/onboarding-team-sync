What's next · RN migration

Auth is moving to Descope

Instead of me building the login flow custom in React Native, the team is integrating Descope, an auth-as-a-service product. The good news: the Descope flow can be configured to match either Flow A or Flow B, so our experiment's conclusions still apply.

  • What changes: auth backend + UI provided by Descope, not custom code.
  • What doesn't: the visual flow we A/B tested can be replicated in Descope.
  • What we keep: Flow B winning conclusion → configure Descope as Flow B.
