Follow-up · Migration tool

Will it scale for beauty?

Not yet — today it's "credential handoff," not full self-serve migration.

  • 4 platforms have API import (Booksy, Square, theCut, Setmore). The rest are CSV / manual.
  • Booksy is automated, but the token is still grabbed by hand — OBS logs in, copies the token, feeds the worker. The app just stores the barber's creds; nothing auto-triggers.
  • The 1-day spike tests if the backend can auto-login → get the token → fire the worker on its own (self-serve). Risk: if Booksy uses captcha / device verification, it's "not feasible without a headless browser" and we stay manual.
  • Beauty has no API integration yet (GlossGenius, Vagaro, Fresha, MyTime) — auto-login alone won't unblock it; we'd build per-platform clients first.
⚠ The real GA blocker

OBS turnaround — haven't checked it. If they answer in ~24h that buys time; if it stretches, that's the blocker, not the automation.
